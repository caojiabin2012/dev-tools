# 剪切板管理

## 功能说明

自动监控系统剪切板，保存文本和图片的复制历史，支持搜索、筛选、置顶、复制回剪切板；**v0.4.0 起支持 GIF 动图**采集与预览。

## 数据库设计

```sql
CREATE TABLE clipboard_items (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    content_type TEXT NOT NULL,        -- 'text' | 'image'
    content_text TEXT,
    content_image BLOB,                -- PNG 或 GIF 原始二进制
    image_width INTEGER,
    image_height INTEGER,
    mime_type TEXT,                     -- image/png | image/gif | ...
    ...
);
```

## 图片采集流程 (monitor.rs)

优先级从高到低：

1. **文件列表**（资源管理器复制 `.gif`）→ `arboard` file_list + `read_gif_from_path`
2. **HDROP 兜底** → `clipboard-win` 读取文件路径
3. **剪贴板 GIF 格式** → 扫描 `GIF` 等格式及嵌入 GIF 魔数
4. **静态位图** → `arboard.get_image()` → PNG（RGBA 直接存储，**不做 BGRA 转换**）

若剪贴板上是 `.gif` 文件但读取失败，**不会**用 `get_image()` 的静态缩略图覆盖。

## GIF 支持 (image_io.rs)

| 函数 | 说明 |
|------|------|
| `try_read_gif_from_file_list_paths` | 从复制文件路径读原 GIF |
| `try_read_gif_from_hdrop` | HDROP 兜底 |
| `try_read_gif_from_clipboard_formats` | 剪贴板格式扫描 |
| `set_clipboard_gif` | 写回 GIF + PNG 首帧 |
| `extract_gif_payload` | 从容器数据中提取 `GIF87a`/`GIF89a` |

写回 GIF 时使用 `set_clipboard_gif`；普通 PNG/JPEG 仍走 `arboard.set_image`。

## 图片预览 (v0.4.0)

- 命令 `get_clipboard_image_data_url` 返回 `data:image/gif;base64,...` 或 PNG/JPEG
- 前端 `ClipboardImage` 组件加载 data URL，**GIF 可播放动画**
- 列表/详情：`mime_type === 'image/gif'` 时显示「GIF」标签

## 颜色修复

arboard 返回的已是 **RGBA** 格式。早期误做 BGRA→RGBA 转换导致红蓝通道互换（偏色），已移除。

## 全局状态 (mod.rs)

```rust
pub static LAST_CLIPBOARD_HASH: Lazy<Arc<Mutex<String>>> = ...;
pub static SKIP_NEXT_IMAGE: AtomicBool = AtomicBool::new(false);
```

写回图片/GIF 前设置 `SKIP_NEXT_IMAGE`，防止监控重复捕获。

## API 命令

| 命令 | 说明 |
|------|------|
| `get_clipboard_items` | 分页列表 |
| `get_clipboard_item_detail` | 详情（含图片字节） |
| `get_clipboard_image_data_url` | Base64 Data URL 预览（GIF 动画） |
| `copy_image_to_clipboard` | 复制图片/GIF 到系统剪贴板 |
| `ocr_image` | OCR（详情页 OCR 入口已注释，保留命令） |
