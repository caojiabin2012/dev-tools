# 版本升级与发布

本文记录 **从改版本号 → 推送 GitHub → 打 tag → 触发 CI 打包** 的完整步骤与命令（以 `v0.4.11` 为例）。

**仓库**：<https://github.com/caojiabin2012/dev-tools>  
**默认分支**：`master`  
**CI 触发**：推送 `v*` 格式 tag（`.github/workflows/release.yml`）  
**发版说明模板**：见 [13.release.md](./13.release.md)

---

## 0. 前置条件

```powershell
cd d:\jiabin\dev-tools

# 确认 Git 身份（须为 cao.jiabin，不能是 cursoragent）
git config user.name
git config user.email

# 确认远程仓库
git remote -v
# 应指向 git@github.com:caojiabin2012/dev-tools.git

# 确认当前分支
git branch --show-current
# 应为 master
```

若远程不对：

```powershell
git remote set-url origin git@github.com:caojiabin2012/dev-tools.git
```

## 1. 修改版本号（0.4.10 → 0.4.11）

以下 **4 处必须一致**：

| 文件 | 字段 |
|------|------|
| `package.json` | `"version"` |
| `src-tauri/Cargo.toml` | `[package].version` |
| `src-tauri/tauri.conf.json` | `"version"` |
| `src-tauri/Cargo.lock` | 根包 `DevTools` 的 `version`（见下一步自动同步） |

改完 `Cargo.toml` 后同步 lock 文件：

```powershell
cd src-tauri
cargo check
cd ..
```

### CHANGELOG

在根目录 `CHANGELOG.md` 顶部追加一节，格式参考历史条目。

## 4. 提交并推送到 GitHub

```powershell
cd d:\jiabin\dev-tools

git status

  src/ src-tauri/src/

```

推送 `master` 后，若已配置 `GITEE_SSH_KEY`，`.github/workflows/sync-gitee.yml` 会自动同步 Gitee。

## 5. 打 tag 触发打包

**tag 名必须为 `v` + 版本号**：

```powershell
git tag -a v0.4.11 -m "v0.4.11"

git push origin v0.4.11
```

推送 tag 即触发 Release Build 工作流。

## 6. 查看 CI 是否成功

### 网页

- Actions：<https://github.com/caojiabin2012/dev-tools/actions/workflows/release.yml>
- Releases：<https://github.com/caojiabin2012/dev-tools/releases>

确认 `build-windows`、`build-linux`、`build-macos`、`release` 均通过。

### GitHub CLI

```powershell
gh run list --repo caojiabin2012/dev-tools --workflow release.yml --limit 5

gh run watch --repo caojiabin2012/dev-tools

gh release view v0.4.11 --repo caojiabin2012/dev-tools
```

失败时查看日志：

```powershell

```

## 9. 常见问题

| 问题 | 处理 |
|------|------|
| CI 未触发 | tag 必须以 `v` 开头，如 `v0.4.11` |
| Release 创建失败 | 检查 `release-notes/vX.Y.Z.md` 是否存在 |
| tag 打错未推送 | `git tag -d v0.4.11` 后重建 |
| tag 已推送需作废 | `git push origin :refs/tags/v0.4.11` 再删本地 tag（慎用） |
| 仅本地打包 | `pnpm tauri build`，产物在 `src-tauri/target/release/bundle/` |
| 更新弹窗只显示 GitHub 链接 | CI 生成的 `latest.json` 里 `notes` 须写入 `release-notes/vX.Y.Z.md` 正文；已发版可重新上传 `latest.json`（见下） |

### 修复已发布版本的更新说明（latest.json）

应用内「更新内容」读取 Release 附件 `latest.json` 的 `notes` 字段，**不是** GitHub Release 页面正文。若只显示 `See https://github.com/...` 链接，说明当时 CI 未嵌入发版说明。

**方式 A（推荐）**：合并 CI 修复后，在 GitHub Actions 对该 tag 重新跑 `release` job，或删除 tag 后重打（会重建 Release 资产）。

**方式 B（仅替换 latest.json）**：本地生成并上传：

```powershell
cd d:\jiabin\dev-tools

# 下载当前 latest.json
Invoke-WebRequest -Uri "https://github.com/caojiabin2012/dev-tools/releases/download/v0.4.11/latest.json" -OutFile latest.json.tmp

# 将 release-notes 正文写入 notes（需 Python 3）
python -c "import json,pathlib;p=pathlib.Path('.');j=json.loads((p/'latest.json.tmp').read_text(encoding='utf-8'));j['notes']=(p/'release-notes'/'v0.4.11.md').read_text(encoding='utf-8');(p/'latest.json.fixed').write_text(json.dumps(j,ensure_ascii=False,indent=2),encoding='utf-8')"

gh release upload v0.4.11 latest.json.fixed --clobber --repo caojiabin2012/dev-tools
```

上传后，旧版客户端再次「检查更新」即可看到完整更新说明。
