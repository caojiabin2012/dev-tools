# 关于与更新

## 功能说明

显示应用信息，启动时自动检查 GitHub Release，支持**应用内下载并静默安装**更新（v0.4.1 修复安装流程）。

## 文件

- `src-tauri/src/settings.rs` - `check_for_update` / `download_and_install_update`
- `src/components/settings.tsx` - 设置 → 关于 Tab（更新入口）
- `src/components/about.tsx` - 独立关于页（如有）
- `src/lib/settings-api.ts` - API 封装
- `src/lib/use-update.ts` - 启动时自动检查更新 Hook

## 检查更新

通过 GitHub Releases API：

```
GET https://api.github.com/repos/caojiabin2012/dev-tools/releases/latest
```

版本比较使用语义化逐段比较（`is_newer_version`），非简单字符串相等。

平台安装包优先级：

| 平台 | 后缀 |
|------|------|
| Windows | `-setup.exe` → `.msi` |
| macOS | `.dmg` |
| Linux | `.AppImage` → `.deb` |

## 下载与安装 (v0.4.1)

### 流程

1. 用户点击「立即更新」
2. 后端下载安装包到 `%TEMP%`
3. Windows：`Unblock-File` 解除 Internet 区域拦截
4. 后台启动安装器（NSIS `/S` 或 MSI `/passive`），**不阻塞等待**
5. 约 800ms 后 `app.exit(0)`，释放文件锁供安装器覆盖

### 命令

```rust
download_and_install_update(app: AppHandle, download_url: String) -> Result<String, String>
```

返回：`安装程序已启动，应用即将退出以完成更新`

### 前端

- 设置 → 关于：显示版本、更新说明、立即更新按钮
- 侧边栏：有新版本时显示更新角标
- `useUpdate`：应用启动时静默检查

## CI 发布

- Tag `v*` 触发 `.github/workflows/release.yml`
- 产物上传 GitHub Release + `latest.json`
- 更新说明：`release-notes/vX.Y.Z.md`
