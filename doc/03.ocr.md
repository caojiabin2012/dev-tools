# 图片 OCR 识别

## 功能说明

对剪切板中的图片进行文字识别（OCR），返回全文和每个文字行的坐标区域，支持中英文。

## 文件

- `src-tauri/src/ocr/mod.rs` - `OcrResult`/`OcrRegion` 定义 + 平台分发
- `src-tauri/src/ocr/windows_ocr.rs` - Windows OCR (原生 WinRT API)
- `src-tauri/src/ocr/macos_ocr.rs` - macOS OCR (Vision 框架)
- `src/lib/clipboard-api.ts` - `ocrImage()` API
- `src/components/clipboard/clipboard-detail.tsx` - OCR 按钮

## 数据结构

```rust
/// OCR 识别结果
pub struct OcrResult {
    pub text: String,              // 全文（所有行拼接）
    pub regions: Vec<OcrRegion>,   // 每行文字的坐标区域
}

/// 单行文字的坐标区域（归一化 0-1）
pub struct OcrRegion {
    pub text: String,     // 该行文字内容
    pub x: f64,           // 左上角 x (占图片宽度的比例)
    pub y: f64,           // 左上角 y (占图片高度的比例)
    pub width: f64,       // 区域宽度比例
    pub height: f64,      // 区域高度比例
}
```

坐标归一化为 0-1，前端按实际图片尺寸缩放后绘制高亮框。

## 实现原理

### 平台分发 (mod.rs)

```rust
pub async fn recognize(image_data: &[u8]) -> Result<OcrResult, String> {
    #[cfg(target_os = "windows")]
    { windows_ocr::recognize(image_data).await }

    #[cfg(target_os = "macos")]
    { macos_ocr::recognize(image_data).await }

    #[cfg(not(any(target_os = "windows", target_os = "macos")))]
    { Err("OCR not supported on this platform".to_string()) }
}
```

### Windows (windows_ocr.rs) — 原生 WinRT

使用 `windows` crate 直接调用 WinRT API，不依赖 PowerShell：

```rust
fn run_ocr(path: &Path, image_data: &[u8]) -> Result<OcrResult, String> {
    // 1. 初始化 WinRT（多线程模式，全局只初始化一次）
    ensure_winrt();  // RoInitialize(RO_INIT_MULTITHREADED)

    // 2. 加载图片为 SoftwareBitmap
    //    优先: StorageFile + BitmapDecoder（支持系统解码器）
    //    回退: image crate 解码 → 手动创建 RGBA8 SoftwareBitmap
    let bitmap = load_bitmap(path, image_data)?;

    // 3. 确保像素格式为 Gray8/Bgra8/Rgba8
    let bitmap = ensure_supported_format(bitmap)?;

    // 4. 创建 OCR 引擎（优先用户语言，回退 zh-CN/en-US）
    let engine = create_ocr_engine()?;
    //   TryCreateFromUserProfileLanguages()
    //   → AvailableRecognizerLanguages() 第一个
    //   → zh-CN, en-US 硬编码回退

    // 5. 执行识别
    let ocr = engine.RecognizeAsync(&bitmap).get()?;

    // 6. 构建结果（全文 + 每行坐标）
    build_result(ocr, &bitmap)
}
```

**坐标提取**: 遍历 `OcrLine` → `OcrWord`，合并每行所有 word 的 `BoundingRect`，归一化为 0-1。

**错误提示**: 如果识别结果为空，返回"未识别到文字，请确认系统已安装 OCR 语言包（设置 → 时间和语言 → 语言）"。

### macOS (macos_ocr.rs) — Vision 框架

通过 Swift 调用 Vision 框架，输出 JSON 格式：

```swift
// 1. NSImage → CGImage
// 2. VNRecognizeTextRequest 识别文字
// 3. 收集每个 observation 的 boundingBox（坐标翻转: y = 1 - y - height）
// 4. 输出 JSON: {"text": "...", "regions": [{"text":"...", "x":0.1, "y":0.2, ...}]}
```

Rust 端解析 JSON 为 `OcrResult`。

**识别配置**:
- `recognitionLevel = .accurate`
- `recognitionLanguages = ["zh-Hans", "zh-Hant", "en-US", "en-GB"]`
- `usesLanguageCorrection = true`

### 其他平台

暂不支持，返回错误提示。

## API 命令

| 命令 | 参数 | 返回 |
|------|------|------|
| `ocr_image` | `image_data: Vec<u8>` (PNG 字节) | `OcrResult` (全文 + 坐标) |

## 二次开发

- **添加新语言**: 修改 `create_ocr_engine` 中的回退语言列表 (Windows) 或 `recognitionLanguages` (macOS)
- **添加 Linux OCR**: 可集成 Tesseract (`tesseract-ocr` crate)
- **优化识别**: 可添加图像预处理 (二值化、去噪)
- **修改坐标精度**: 调整 `build_result` 中的归一化逻辑
